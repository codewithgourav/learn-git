Git and Github

Git and GitHub are two technologies that every developer should learn, irrespective of their field.

What is Git

Git is a version control system which lets you track changes you make to your files over time. With Git, you can revert to various states of your files (like a time traveling machine). You can also make a copy of your file, make changes to that copy, and then merge these changes to the original copy.

What is Version Control

The management of changes to documents,computer programs, large websites, and other collections of informations.

What is Github

GitHub is an online hosting service for Git repositories.

Prerequisites/Terms

  • A command line interface.
  • Directory (Folder).
  • cd Change Directory
  • A text editor of your choice (I will be using VS Code).
  • A GitHub account.
  • Repository Project, or the folder/place where your projects are kept.

Git Commands

  • clone Bring a repo that is hosted somewhere like GitHub into a folder on your local machine.
  • add Track your files and changes in Git.
  • commit Save your files in Git.
  • push Upload Git commits to a remote repo, like GitHub.
  • pull Download changes from remote repo to your local machine, the opposite of push.

Workflow

Git Branching

You have already seen by now that previously we did git push origin main. What is main? Main/Master is a naming convention for the main or default branch is a repository. So if you are working on a particular branch your code will remain there. So all the code all the commits will be confined within that particular branch.

This is called branching, because it starts to look like a tree when we start creating seperate banches out of the main branch. So you can make other branches like feature branch. When you create feature branch the code remains exactly the same as in the master/main branch. But as you make updates to feature branch, those changes are only seen in the feature branch. So if you make changes in the feature branch, you make a commit to save those changes on to that branch. And then you switch back to master branch, you won't be able to see any of those changes that you made on that feature branch. Each individual branch has no way of knowing what commits or what changes are made to any other branch. Each branch is only keeping track of what changes are made on its own branch.

Why is this useful? It's extremely useful because you will be building out new features to applications that may break your code and you don't want to save them to the main/master branch. So you want to add those new code seperately and not add/merge it to the main/master branch unless it's correct. And this is really helpful when we have multiple developers working on the same repository.

Merge Conflict

In real life you are not always going to have such an easy time merging. There is something called merge conflicts in git, where basically you're are writing your own code on your own branch. Maybe other people writing their code on their branches. And main branch is getting updated from time to time. So it's possible for multiple people to change the same files.So Git doesn't know which code you want to keep, or which code is redundant, or which code you want to get rid of. That is when you face merge conflict.

Steps To Create Merge Conflict

  • Switch to feature branch and make some changes to index.html file and then commit the change to local.
  • Switch to main branch and make some changes to index.html file and then commit the change to local.
  • Switch to feature branch and type the command git diff main to observe the difference/changes with the main branch.
  • Type the command git merge main. Once you do that you will get merge conflict message. Follow the below image.
  • The last step you have to do is select Accept Both Changes to accept the incoming changes from main and then commit the new changes in feature branch locally.

Undo in GIT

git reset command is used in CLI to undo git add .. To undo a commit we use the comman git reset HEAD~1

Remember git reset --hard command not only undo a commit but also removes the modifications from the code.

Forking

A fork is a copy of a repository that you manage. Forks let you make changes to a project without affecting the original repository. You can fetch updates from or submit changes to the original repository with pull requests.
